WEB-02 Cookie Monster
Medium
Time 00:00
Scenario

CorpX Staff Portal assigns a session cookie after login. The cookie contains your role — and the application trusts whatever role is in the cookie.

You have valid credentials for a standard user account. Your goal is to escalate your privileges to admin by forging a valid admin session cookie and accessing the admin panel.

The cookie is not encrypted. But it is signed. You need to understand how the signature works before you can forge one.

Hints
The cookie has two parts separated by a dot. The first part is Base64 encoded JSON. Decode it to see what fields are inside. Just changing the role in the decoded JSON and re-encoding will not work — the second part is a signature that must also match.
The app loads a JS file. Read it carefully — developers sometimes leave implementation comments. A hex string prefixed with 0x in a comment about session integrity is worth investigating. Hex decode it to get the signing key.
Once you have the secret key and know the algorithm, you can compute the correct signature for any payload. The algo comment in the JS tells you exactly what to compute. In Python: import hashlib; hashlib.md5((payload+key).encode()).hexdigest()[:8]. Build the forged cookie as newpayload.newsig and send it as the session cookie on GET /dashboard.
