Time 00:00
Scenario

The CorpX security team flagged unusual outbound traffic from an internal host. A 60 second packet capture was taken at the network egress point during the suspected window.

Your task is to analyze the capture in Wireshark and determine what data was exfiltrated, and how.

This is a real .pcap file. There is no simulation here — open it in an actual packet analyzer.

Objectives
  • 1Download the capture file
  • 2Open it in Wireshark and identify all protocols present
  • 3Use Follow TCP Stream on the FTP session
  • 4Identify the suspicious DNS query pattern
  • 5Extract and decode the exfiltrated data, submit the flag
Hints
Use the filter bar. Try ftp to isolate the FTP control session — right click any packet in that conversation and choose Follow then TCP Stream to see the full plaintext exchange. Separately, try filtering dns to see every DNS query in the capture.
The FTP login credentials are real plaintext, but they are not the answer to this challenge. They demonstrate a different problem entirely. The actual exfiltration uses a completely different protocol. Look at which DNS queries return NXDOMAIN and ask yourself why a host would repeatedly query a domain that does not resolve.
DNS exfiltration encodes stolen data as subdomain labels because DNS traffic is rarely blocked or inspected. Collect every unique subdomain label queried against the same parent domain, in the order they appear by timestamp. The labels look like Base32 — note that Base32 uses A-Z and 2-7 only, and is case-insensitive on decode. Concatenate the decoded chunks in query order.

Suggested Analysis Path
1
Open in Wireshark, scan protocols
Check Statistics then Protocol Hierarchy to see what is present: TCP, UDP, DNS, FTP. Note the proportion of each.
2
Filter and follow the FTP stream
Apply filter ftp, right click a packet, choose Follow then TCP Stream. Read the full plaintext session.
3
Filter DNS and inspect query names
Apply filter dns. Look at the Info column for each query. Most resolve normally. A few return NXDOMAIN against an unfamiliar parent domain.
4
Collect the unusual subdomain labels in order
Sort by timestamp. Write down each subdomain label queried against the suspicious parent domain.
5
Decode and concatenate
Decode each label, join them in order, and you should have a readable flag string.
Flag Captured
DNS exfiltration uses subdomain labels to smuggle data past firewalls that allow DNS but inspect HTTP. NXDOMAIN responses are normal here since these domains were never meant to resolve.
