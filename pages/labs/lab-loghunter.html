Scenario

On 8 November 2024, CorpX SOC received an alert that the public-facing web server had been actively probed. The security team pulled a full day's Apache access log covering 2,156 requests.

Your job is to sift through the noise, identify the attacker, determine which tool they used to find the target endpoint, and name the path they ultimately attacked. Combine your findings into the flag.

This is an Apache Combined Log Format file. Each line: ip - - [timestamp] "METHOD path HTTP/1.1" status size "referer" "user-agent". The User-Agent field at the end is key for identifying automated tools. Use grep, awk, sort, and uniq -c on Linux/macOS, or any text editor with search on Windows.
Objectives
Identify the attacker's IP address
Determine which scanning tool was used for the SQL injection attack
Find the target path that sqlmap attacked most
Submit the flag in format ACX{ip__tool__path}
Analysis Path
1
Get a bird's eye view of the traffic
Count requests per IP: awk '{print $1}' corpx_access.log | sort | uniq -c | sort -rn | head -20. One IP stands out with a high request count compared to all others.
2
Isolate traffic from the suspicious IP
Filter all lines from that IP: grep "ATTACKER_IP" corpx_access.log. Look at the User-Agent column (last field). You will see two distinct tools, each active at different times.
3
Identify the directory brute-force phase
One tool made dozens of rapid sequential requests, mostly returning 404, probing for directories and files. Note the User-Agent string and what paths returned 200 or 403.
4
Identify the SQL injection phase
A second tool appeared later, hitting one specific path repeatedly with query strings containing SQL payloads. The User-Agent identifies the tool by name and version.
5
Extract the three values and build the flag
You need: attacker IP, name of the SQL injection tool (lowercase, no version), and the base path without query string. Format: ACX{ip__toolname__/path/file.php} (double underscore between each).
Hints
Run awk '{print $1}' corpx_access.log | sort | uniq -c | sort -rn | head -5. The attacker's IP will have significantly more requests than any normal visitor. Normal users make 1 to 10 requests. The attacker made over 100.
Filter by attacker IP and look at unique User-Agents: grep "45.x.x.x" corpx_access.log | awk -F'"' '{print $6}' | sort -u. You will see exactly two non-browser strings. One is a directory brute-forcer, the other is a SQL injection framework. The flag uses the SQL injection tool name only.
Filter by the sqlmap User-Agent: grep "sqlmap" corpx_access.log | awk '{print $7}'. All sqlmap requests go to the same base PHP file. Strip the query string and use only the path portion in the flag. The path starts with /admin/.
