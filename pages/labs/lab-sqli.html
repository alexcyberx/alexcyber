Scenario

A corporate internal portal was found exposed on the network during a red team engagement. The login form passes user-supplied input directly into a SQL query with no sanitization. Your goal is to bypass authentication and extract the flag from the secrets table. Use Burp Suite to intercept and modify the login request.

Objectives
  • 1
    Identify the injectable parameter in the login request
  • 2
    Cause a SQL error to confirm injection is possible
  • 3
    Craft a payload that bypasses authentication as admin
  • 4
    Extract the flag from the secrets table

Vulnerable Code
// server-side — Node.js + SQLite const query = `SELECT * FROM users WHERE username = '${username}' AND password = '${password}'`; db.prepare(query).all();
Target Endpoint
POST /api/lab/web01/login Host: TARGET_HOST Content-Type: application/x-www-form-urlencoded username=admin&password=test
Hints
The login query is built by directly embedding your input into a SQL string. A single quote character ' in your input will break the query syntax. Send a single quote as the username and observe the server response carefully — if you get a database error back, you have confirmed the injection point.
SQL injection in a WHERE clause lets you change the logic of the condition. Think about how you can make the WHERE clause always evaluate to true regardless of what the password column contains. SQL comments can also be useful — they let you ignore the rest of the query after your injection. The goal is to match an admin-role user without knowing their real password.
Your payload goes in the username field. You want to close the opening single quote, inject a condition that makes the WHERE clause true for a specific user, then comment out the remaining SQL so the password check is never evaluated. Consider targeting a user whose role you already know from the schema. The flag will only appear in the response if the bypassed account has the admin role.
