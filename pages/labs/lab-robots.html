Scenario

You are doing recon on CorpX Backup Manager. The server has a robots.txt file that discloses several internal paths.

Most are decoys. One leads to a real directory listing containing a backup archive. The archive is encrypted — but the JS file loaded by the application contains a clue about the encryption scheme.

Enumerate, find the file, extract the key, decrypt, get the flag.

Hints
robots.txt lists multiple disallowed paths. Test each one — most will return 403 or 503. A real directory listing returns 200 with HTML content showing filenames and sizes. Try all of them, not just the first one.
The directory contains multiple files — a log, a large sql.gz, and a small .dat file. The sql.gz has been rotated. The .dat file is 58 bytes and contains hex-encoded encrypted data. The encryption scheme and key are documented in the JS file loaded by the app.
XOR is symmetric — the same operation decrypts as encrypts. In Python: key="..."; enc=bytes.fromhex("..."); print(''.join(chr(b^ord(key[i%len(key)])) for i,b in enumerate(enc))). The key is hex-encoded in the JS comment — hex decode it first to get the actual key string.
